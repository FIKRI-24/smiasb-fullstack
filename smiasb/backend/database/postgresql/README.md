# PostgreSQL Schema SMIASB

Folder ini berisi persiapan schema PostgreSQL hasil konversi dari dump MySQL:

`../smiasb_db_mysql_backup.sql`

Tahap ini hanya membuat struktur database. Aplikasi SMIASB masih memakai koneksi MySQL sampai tahap migrasi koneksi dilakukan.

## File

- `schema.sql`: schema PostgreSQL untuk 15 tabel SMIASB, termasuk primary key, unique key, index, foreign key, check constraint, dan helper reset sequence.

## Cara Menjalankan Schema

Pastikan sudah berada di database PostgreSQL kosong, lalu jalankan:

```bash
psql -U postgres -d smiasb_postgres -f smiasb/backend/database/postgresql/schema.sql
```

Contoh jika menjalankan dari folder `smiasb/backend/database/postgresql`:

```bash
psql -U postgres -d smiasb_postgres -f schema.sql
```

## Catatan Konversi

- `AUTO_INCREMENT` dikonversi ke `GENERATED BY DEFAULT AS IDENTITY`.
- `INT` dikonversi ke `INTEGER`.
- `BIGINT UNSIGNED` dikonversi ke `BIGINT`.
- `TINYINT(1)` dikonversi ke `BOOLEAN`.
- `DATETIME` dan `TIMESTAMP` dikonversi ke `TIMESTAMP`.
- `LONGTEXT` dan `MEDIUMTEXT` dikonversi ke `TEXT`.
- `ENUM` dikonversi menjadi `VARCHAR` dengan `CHECK`.
- Kolom JSON-like tetap memakai `TEXT` agar kompatibel dengan backend lama.
- Syntax khusus MySQL seperti `ENGINE`, `CHARSET`, `COLLATE`, backtick, `UNSIGNED`, dan `ON UPDATE CURRENT_TIMESTAMP` tidak dipakai.

## Setelah Import Data

Jika data MySQL diimport dengan nilai `id` eksplisit, jalankan blok reset sequence yang ada di bagian bawah `schema.sql`. Blok itu sengaja dikomentari agar tidak berjalan sebelum data benar-benar diimport.

## Catatan Penting

Schema ini belum mengubah koneksi aplikasi, route, endpoint, validasi, role access, import Word/Excel, chatbot, assessment, atau laporan. Tahap berikutnya perlu menyiapkan konversi data dan penyesuaian query backend dari MySQL ke PostgreSQL.
